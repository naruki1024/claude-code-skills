# スキルの基本構造

## ディレクトリ構成

スキルは以下の構造で構成される:

```
.claude/skills/{skill-name}/
├── SKILL.md                # 必須: スキルの定義とワークフロー
├── agents/                 # オプション: Task仕様書
│   ├── {task-name}.md
│   └── ...
├── references/             # オプション: 外部化された知識
│   ├── {topic}.md
│   └── examples/           # 具体例は別フォルダに
│       └── ...
└── scripts/                # オプション: 自動化スクリプト
    └── ...
```

## 配置場所による違い

| 配置場所 | 用途 | 利用範囲 |
|----------|------|----------|
| `~/.claude/skills/` | 個人用スキル | 全プロジェクト |
| `.claude/skills/` | プロジェクトスキル | 該当プロジェクトのみ |

## 各ディレクトリの役割

### SKILL.md
- スキル全体の定義と入り口
- ワークフローの流れを記述
- **段階的ロードの起点**となる

### agents/
- 各Taskの仕様書を配置
- 「役割・入力・出力・制約」を記述
- **知識は持たせず、参照パスのみ記載**

### references/
- ルール、テンプレート、サンプルなどの知識
- 1ファイル1トピックで分解
- Taskが必要なときだけ読み込む

### scripts/
- LLMにやらせると非効率な定型処理
- ファイル操作、整形、変換など
- 実行結果は確定的で再現性が高い
