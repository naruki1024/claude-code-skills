# 変更注釈のフォーマット

## 基本形式

変更箇所の直後に HTML コメント形式で注釈を追加する。

```markdown
<!-- {変更種別}: {議事録ファイル名} -->
```

## 変更種別

| 種別 | 用途 | 例 |
|------|------|-----|
| `変更` | 既存仕様の修正 | `<!-- 変更: 20260106_定例.md -->` |
| `追加` | 新規仕様の追加 | `<!-- 追加: 20260106_定例.md -->` |
| `削除` | 機能・仕様の削除 | `<!-- 削除: 20260106_定例.md -->` |
| `スコープ変更` | MVP対象/対象外の変更 | `<!-- スコープ変更: 20260106_定例.md -->` |
| `明確化` | 曖昧な仕様の明確化 | `<!-- 明確化: 20260106_定例.md -->` |

## 配置ルール

### 1. 変更箇所の直後に配置

```markdown
~~旧仕様~~

新仕様

<!-- 変更: 20260106_定例.md -->
```

### 2. 複数項目を一括変更した場合

まとめて1つの注釈:

```markdown
~~項目A~~
~~項目B~~
~~項目C~~

新項目A
新項目B

<!-- 変更: 20260106_定例.md -->
```

### 3. 異なる議事録からの変更が近接する場合

それぞれに注釈:

```markdown
~~古い仕様1~~

新仕様1

<!-- 変更: 20251215_定例.md -->

~~古い仕様2~~

新仕様2

<!-- 変更: 20260106_定例.md -->
```

## 注釈の粒度

### 良い例

```markdown
* ~~**d払い**: 実装予定~~
* **d払い**: MVP対象外（審査待ち）

<!-- スコープ変更: 20260106_定例.md -->
```

### 悪い例（注釈が多すぎる）

```markdown
* ~~**d払い**: 実装予定~~ <!-- 削除: 20260106_定例.md -->
* **d払い**: MVP対象外 <!-- 追加: 20260106_定例.md --> （審査待ち） <!-- 追加: 20260106_定例.md -->
```

## HTMLコメントの注意点

- マークダウンプレビューでは非表示
- Git diff では表示されるため、履歴追跡が可能
- コメント内に `--` を含めない（HTML仕様違反）
