# 反映記録（reflection-log.md）のフォーマット

## ファイルの目的

`/docs/reflection-log.md` は以下を管理する:

1. どの議事録が反映済みか
2. どのファイル・セクションに反映されたか
3. いつ反映されたか

これにより、二重反映の防止と変更履歴の追跡が可能になる。

## ファイル構造

```markdown
# 議事録反映記録

このファイルは `/docs/temp/` の議事録から仕様書への反映履歴を管理します。

---

## 20260106_定例.md

| 反映先ファイル | セクション | 変更種別 |

- d払いをMVP対象外に変更
```

## 各セクションの説明

### 見出し（##）

議事録のファイル名をそのまま使用。

```markdown
## 20260106_定例.md
```

### 反映日時

実際に反映作業を実行した日時。ISO 8601 形式（YYYY-MM-DD HH:MM）を推奨。

```markdown
**反映日時**: 2026-01-09 14:30
```

### 反映先テーブル

| カラム | 説明 |
|--------|------|
| 反映先ファイル | `docs/` からの相対パス |
| セクション | 見出し名またはセクション番号 |
| 変更種別 | 追加/修正/削除/スコープ変更/明確化 |

### 変更概要

箇条書きで簡潔に。1項目1行。

## 順序

**新しい記録が上**になるよう、ファイルの上部に追記していく。

## 追加反映の場合

同じ議事録から追加で反映が行われた場合:

```markdown
## 20260106_定例.md（追加反映）

**反映日時**: 2026-01-10 09:00

| 反映先ファイル | セクション | 変更種別 |
|--------------|----------|---------|
| 02.design/DB/パスワーク_DB設計.md | テーブル定義 | 修正 |

**変更概要**:
- 初回反映で漏れていたDB設計への反映

---

## 20260106_定例.md

**反映日時**: 2026-01-09 14:30
...
```
